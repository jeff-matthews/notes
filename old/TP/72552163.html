Tech Pubs : Set Up Basic Authentication (Postman)

Created by Jeff Matthews, last modified on Mar 02, 2017

Our Payment Gateway API reference uses cURL to show example API request calls. All requests are authenticated using HTTP Basic Authentication. To authenticate, you provide either your test- or live-mode secret key in the HTTP Authorization header. 

Using the cURL command line tool used in the API examples, you set this header using the --user flag, followed by the secret_key and then a trailing colon : While it's fine to use cURL to interact with the API, it's easier to use a client like Postman. Setting up authentication isn't obvious, however, so follow these instructions: 

Step-by-step guide

  1. Download the Postman Chrome plugin and launch the app. 
  2. Next, you need to gather some information Postman needs to authenticate with the payment gateway.
    1. Log in to the AffiniPay web app to retrieve your secret_key.
      1. Developers > Authorized Applications > Your application > Show Credentials
    2. Go to an Online Base64 Encoder.
    3. Paste your secret_key in the text box followed by a colon : 
    4. Click Encode.
    5. Copy the encoded result.
  3. Go back to the Postman app and click  Authorization. 
  4. Select No Auth.
  5. Click Headers.
  6. In the first key text field, enter "Authorization".
  7. In the first value text field adjacent to the key text field, enter "Basic" followed by a space and then paste the Base64-encoded result from step 4.

That's it! You should be able to make calls to the API using your Payment Gateway credentials (i.e., {{secret_key}}).
