Goals

  • Create a requirements document to replace the existing developer landing page at https://developers.affinipay.com. 

  • Conduct research on at least three competitors and write a report. 

  • Demo a working developer landing page prototype.

Background and strategic fit

Most developers have difficulty differentiating between the integration types we support, so helping them decide which type is best for them is an opportunity to enhance the developer experience and be competitive in the market. We should also promote all available documentation resources on the landing page so that visitors have a full understanding of the options available to them (e.g., API reference and tutorials).

The design of our current landing page is misleading because it implies that we only provide two guides; one for third-party app integrations and another for website payment forms. We expect users to discover the other documentation resources we offer on their own by exploring the table of contents or by searching the site.

Assumptions

  • Developers are the primary audience 
  • Developers have a basic understanding of web development and payment processing technologies

TP-IDR - Integration paths

As a developer, I want to see what type of integration methods are available so that I can determine which path is right for me. I also want help deciding which integration method is right for me if I don't already know.

Create an interactive experience based on a binary decision tree to guide undecided developers. For example, a developer clicks "Help me decide which integration is right for me" and we present them with two options. After they click an option, we can present them with another set of options until we've determined the best integration path based on their inputs. From there, we can drop them on the diagram (TP-110) with an integration path already selected so they can see a system overview.

Must have
TP-110IDR - Integration paths diagram

As a partner or developer exploring the AffiniPay platform, I want to see how the system works at a high level so I can determine which type of integration is best for me.

For example, show me a diagram of the client- and server-side interactions involved in a credit card charge from beginning (authorization) to end (capture & settlement) for all integration types. Make the diagram interactive so that clicking a type highlights the system path/touch points for that type with a brief description and grays out the rest.
