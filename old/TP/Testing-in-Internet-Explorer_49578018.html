Tech Pubs : Testing in Internet Explorer

Created by Jeff Matthews, last modified on Feb 01, 2017

Inexplicably, some people still use IE to surf the internets. To ensure major doc site changes are compatible with IE, you should set up a Windows VM for testing. I use Vagrant, but you could easily do this using VirtualBox. These are instructions for using Vagrant on Mac OS X and Linux.

  1. Install Vagrant.
  2. Install VirtualBox.
  3. Find a Windows 10 box in the HashiCorp Atlas directory.
  4. Create a local directory for this box and navigate to it: 
    mkdir win7 && cd win7
  5. Initialize Vagrant:
    vagrant init

  6. Edit the VagrantFile in your directory. It should look like this:

    Vagrant.configure(2) do |config|
        config.vm.box = “win7/ie11”
        config.vm.communicator = "winrm"
        config.vm.network "forwarded_port", host: 3389, guest: 3389
        config.vm.provider "virtualbox" do |v|
             v.cpu = 2
             v.memory = 2048
        end
    end


  7. Start Vagrant (you may need to open your VirtualBox client to see the VM): 
    vagrant up --provider virtualbox 
  8. Build and serve the doc site locally as normal, but bind the server to 0.0.0.0 instead of the default loopback address so you can access the server from your VM: 
    jekyll build && jekyll serve -H 0.0.0.0
  9. Open IE on your VM and enter your development machine's IP address followed by the port number (e.g., http://10.1.188.84:4000).
