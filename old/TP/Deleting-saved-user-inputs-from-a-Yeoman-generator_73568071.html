Tech Pubs : Deleting saved user inputs from a Yeoman generator

Created by Jeff Matthews, last modified on Apr 05, 2017

Yeoman let's you save user inputs as default values by setting the store attribute to true.

var prompts = [{
name: 'authorName',
messsage: 'Author\'s Name',
store: true
}];

When a user enters their name, the generator returns that name as a default answer the next time they run the generator. These settings are saved in the user's .yo-rc-global.json file. For example,

{
  "generator-make-readme:1.0.5": {
    "promptValues": {
	  "authorName": "Jeff Matthews"
    }
  }
}

When testing, sometimes it's preferable to delete saved inputs:

  1. Open your local /Users/<user>/.yo-rc-global.json file and delete the line that contains the saved input (e.g., "authorName": "Jeff Matthews").
