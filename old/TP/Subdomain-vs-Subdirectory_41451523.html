Tech Pubs : Subdomain vs Subdirectory

Created by Jeff Matthews, last modified on Aug 11, 2016

According to Matt Cutts (Head of Google Webspam team), Google's Page Rank algorithm gives root domains and subdomains "roughly" equal weight where SEO is concerned. For example, https://affinipay.com/developers and https://developers.affinipay.com are treated "roughly" the same. I found articles disputing that statement from SEO aficionados, but I think the bottom line for us is to use subdomains for the following reasons:

  • Stephen prefers the clarity that the subdomain invokes for developer documentation.
  • The DNS subdomain and web server to host files is already configured and ready to use. 
  • Since we're using separate systems to generate and manage content on the main site and developer doc site (Wordpress & Jekyll respectively), it may not be practical or technically possible to use a subdirectory. We know there's no conflict using a subdomain in this regard; we can't say the same for subdirectories without further investigation.
  • Even if using subdirectories is technically feasible, it will most likely complicate the release management process for doc updates (e.g., introducing risk to our main sites). Using a subdomain enables us to deploy doc updates to production on an ad hoc basis, which is important, without impacting the availability of our main website(s).
  • Using a subdomain gives us an opportunity to establish, cultivate, and control a "developer" brand that may not otherwise be accommodated through normal marketing channels.
  • Subdirectory leads to bad UX. The new doc site design differs greatly from the main site.
