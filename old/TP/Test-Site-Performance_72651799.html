Tech Pubs : Test Site Performance

Created by Jeff Matthews, last modified on Mar 07, 2017

Running a site through Google's PageSpeed Insights tool is a handy way to check how performant it is. Just enter a web URL in the text box and click Analyze. To test how local development changes impact performance, you'll need to expose your local web server to the internet. There are lots of tools to help you do that securely, but we're going to use ngrok in this guide.

This article assumes you know how to build the dev-doc site. You have to do that first before proceeding with this guide.

Ngrok

  1. Download ngrok.
  2. Unzip the archive to a directory (e.g., Downloads).
  3. Open a terminal and navigate to the directory where you unpacked the ngrok archive (e.g., cd ~/Downloads).

  4. Expose your local development web server to the internet with this command; where <port#> is the port your local web server is listening on (e.g., http://localhost:3000/): ./ngrok http <port#>

  5. Copy one of the Forwarding URLs:

  6. Go to Google's PageSpeed Insights tool.
  7. Paste the Forwarding URL in the text box and click Analyze.

Special Test Cases

If you're trying to test specific things, like minified HTML/CSS/JS assets, you need to follow special build instructions:

  1. Build the site and execute all minification tasks: gulp build-prod
  2. Start a local web server and serve the site: gulp serve

Now you're ready to fire up ngrok and test the site.

Running any other build method (e.g., rake, gulp build-dev) will overwrite all file changes executed by gulp build-prod and gulp serve because Jekyll's default serve behavior is to recompile the site right before serving it.
