Goals

  • Design and publish API reference content that's easy to use, maintain, and scale
  • Migrate remaining content from https://chargeio.com/developers to new developer doc site
    • Redirect https://chargeio.com/develoeprs to https://developer.affinipay.com
    • Replace references to chargeIO in the VT UI and https://affinipay.com 
  • Redesign landing page to strengthen the AffiniPay developer brand

Background and strategic fit

This phase will conclude our efforts to consolidate content from https://affinipay.com/developer and https://chargeio.com/developers into a new web site (https://developer.affinipay.com). It will also allow us to begin building a developer-specific brand that we can promote to our partners.

Requirements

#TitleUser StoryImportanceNotes
1Content Migration

As a developer, I want to access all Virtual Terminal (VT) or AffiniPay Payment Gateway developer documentation on a single site so that I don't have to go to multiple sources for integration resources (e.g., guides, tutorials, sample code, client libraries, API reference, etc.).

Must have

TP-26 & TP-46

2API Reference Design

As a developer, I want to be able to browse AffiniPay's API reference material so that I can easily identify:

  • All available APIs
  • All API endpoints associated with a selected API
  • HTTP operations available for each endpoint
  • Object types, including property descriptions, associated with each API
  • Webhooks, including events and event types, associated with each API
  • Messages and error codes associated with each API
  • Sample requests for each API endpoint operation
    • Stretch goal/Nice to have: I want to be able to toggle between cURL, PHP, Ruby, and JavaScript samples
  • Sample responses for each API endpoint operation
Must haveTP-44 & TP-47
3Landing Page Redesign

As a developer or manager/decision maker visiting https://developer.affinipay.com for the first time, I want the landing page to answer the following questions so that I can quickly assess how AffiniPay can help me or my company:

  • What is AffiniPay?
  • What brands represent AffiniPay's line of payment solutions?
  • What type of integrations does AffiniPay support?
  • What type of integration is right for me and how do I implement it?
  • Who are AffiniPay's current integration partners?
