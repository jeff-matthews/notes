Goals

  • Create first-class documentation for every AffiniPay Github repo
  • Update and publish standardized READMEs for all active repos
  • Establish guidelines for README authors 
  • Create a Yeoman generator to more easily generate future READMEs according to common conventions and best practices

Background and strategic fit

Github is an important platform for collaborative software development. It's also a channel for AffiniPay to share tools with our partner/developer community. We need to make sure that the current tools/plugins work and provide accurate documentation.

Assumptions

  • Tech Pubs doesn't own repos; just the READMEs

User interaction and design

Every repo should adhere to the following guidelines:

  • Every repo that contains code also contains a README
  • Every README follows a standardized information architecture based on common conventions and best practices
  • Every README concisely and accurately explains how to use the library/repo
  • Every README contains relevant, usable code examples except when examples may be impractical
  • Every README contains adequate reference information explaining all methods/functions
