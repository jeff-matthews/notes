Goals

Provide partners/developers/business users with conceptual documentation necessary for understanding what the following new features are and how they're related:

  • Businesses
  • Products
    • Setup fee
    • Trial period
    • Flexible scheduling
    • Multiple billable components (e.g., fixed fees, usage-based fees, quantity based)
  • Subscription management with flexible, per-customer override capability
  • Subscription invoicing
    • PDF generation
    • Email delivery
    • Auto-pay support
  • Subscription notifications (e.g., failed payment)
  • Invoicing and payment history

Publish Product Billing API documentation when we release the feature (overview, reference, and tutorial)

Background and strategic fit

Subscriptions are the first layer of the billing and invoicing feature work currently under development. As we ship new features, we need to publish developer documentation. The scope of this feature is documented in the Product Billing API wiki. This document specifies the scope of developer doc requirements.

Assumptions

  • Docs should be published to https://developers.affinipay.com
