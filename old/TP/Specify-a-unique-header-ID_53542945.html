Tech Pubs : Specify a unique header ID

Created by Jeff Matthews, last modified on Feb 01, 2017

We use kramdown syntax in the /dev-docs repo. By default, Jekyll parses markdown/kramdown formatted headers and automatically prepends a header ID to the compiled HTML element. The default header ID is the same as the header text, only Jekyll substitutes spaces with hyphens.

Example:

# Level 1 Heading ===> <h1 id="level-1-heading">Level 1 Heading</h1>

Kramdown supports explicitly setting header IDs (taken from PHP Markdown Extra and Maruku). If you follow the header text with an opening curly bracket (separated from the text with at least one space), a hash, the ID and a closing curly bracket, the ID is set on the header.

Example:

# Level 1 Heading {#unique-header-id} ===> <h1 id="unique-header-id">Level 1 Heading</h1>
