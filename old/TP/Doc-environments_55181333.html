Tech Pubs : Doc environments

Created by Jeff Matthews, last modified on Mar 02, 2017

There are three environments where we deploy the /dev-docs site:

  • DEV - This where we deploy builds of feature/bug branches for internal review and other testing shenanigans that we want to keep off master.
  • QA - This is where we deploy builds of the master and qa branches after integrating features and bugs. We can also deploy review builds here when necessary for internal stakeholder review.
  • PROD - This is where we deploy builds of the production branch after we merge master.

DEV

The purpose of this environment is to get stakeholders to review content for technical accuracy before merging a feature/bug into master. We can also use this environment to test other site-wide changes. You must manually deploy build archives to this environment because no cron jobs are running.

QA

The purpose of this environment is to formally test features/bugs that have been merged into the master branch. Merges to master kick off a Jenkins build. You must manually deploy build archives to this environment because no cron jobs are running.
